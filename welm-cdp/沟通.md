# welm-cdp 设计演化记录（2026-06-22 完整版）

今天是最近一段时间变化最大的一天之一。

表面上看：

```txt
chrome.js
wait.js
runtime.js
screenshot.js
```

都改了很多。

但本质上，今天真正发生的是：

```txt
cdp/
开始从项目内部代码

演化成

可独立迁移的能力模块
```

---

# 一、今天最核心的设计目标

用户明确提出：

```txt
尽可能让 cdp 目录独立

与目录外无关

未来可以轻松复制到其它项目
```

形成新的判断标准：

对于：

```txt
cdp/*.js
```

都应该问：

```txt
如果把这个文件复制到新项目

还需要复制什么？
```

理想答案：

```txt
只需要：
cdp/*.js

不需要：
config.js
error.js
protocol.js
cmd/*
flow/*
```

这是今天所有重构的根源。

---

# 二、chrome.js 重大收敛

## target.js 被删除

之前：

```txt
chrome.js
    ↓
target.js
```

经过反复讨论：

```txt
Target
到底是谁的领域？
```

最终发现：

```txt
Target
只是 CDP 协议概念

不是用户领域
```

用户真正关心：

```txt
Chrome
Page
Tab
```

不关心：

```txt
Target.createTarget
Target.activateTarget
Target.closeTarget
```

因此：

```txt
target.js
删除
```

Target 退化为：

```txt
chrome.js 内部实现
```

# 三、chrome.js 不再依赖 client.js

今天一个非常重要的认知：

```txt
CDP(...)
```

并不一定属于：

```txt
client pool
```

例如：

```txt
waitPageReady()
```

里面创建的 client：

```js
const client = await CDP(...)
```

只是为了：

```txt
监听 Page.loadEventFired
```

任务完成立即关闭。

因此：

```txt
谁创建
谁关闭
```

成为共识。

---

## waitPageReady 最终方案

历史问题：

```txt
about:blank
↓
loadEventFired
↓
开始监听

错过事件
```

导致超时。

最终方案：

```txt
监听 loadEventFired
↓
waitLeaveAboutBlank()
↓
等待 loadEventFired
```

---

## 三类 Timeout 最终定型

Chrome Ready：

```txt
15s
```

```js
timeout = 15000
interval = 200
```

---

Leave About Blank：

```txt
2s
```

```js
timeout = 2000
interval = 50
```

---

Page Ready：

```txt
30s
```

```js
timeout = 30000
```

# 四、能力层摆脱 config.js

```txt

wait.js

↓

```

---

今天形成新的原则：

```txt

默认值允许重复
```

```txt

```

必须统一。

```js

```

属于：

```txt

```

允许重复。

---

目标：

```txt

直接能跑
```

而不是：

```txt
还要复制 config.js
```

# 六、poll 的重大升级

这是今天第二大的架构变化。

---

## 之前

```js
poll()
```

内部写死：

```js
isPollMatched()
```

本质：

```txt
轮询
+
判断
```

耦合。

---

## 现在

新增：

```js
options.matcher
```

形成：

```js
const matcher =
  options.matcher ?? defaultMatcher;
```

---

poll 变成：

```txt
轮询引擎
```

只负责：

```txt
evaluate
sleep
timeout
```

---

判断逻辑：

```txt
matcher
```

负责。

---

## defaultMatcher 最终规则

```txt
null        false
undefined   false
false       false
0           false
""          false

其它值      true
```

---

## matchTimes

新增：

```js
matchTimes
```

支持：

```txt
连续命中 N 次
才算成功
```

例如：

```txt
防止瞬时状态
防抖
稳定检测
```

```txt

```
