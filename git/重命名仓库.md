好，我给你一套**一次性可执行、低风险、可回滚的完整改名流程**。  
目标：

```text
仓库名：welmcli
本地目录：welmcli
命令：welm
（可选兼容：clichatgpt → welm）
```

---

# 🧭 总体策略（先看这个）

- **先改 GitHub 仓库名（有自动重定向）**
    
- **再改本地目录名（不会影响 Git）**
    
- **最后改命令与代码（可逐步过渡）**
    
- **全程可回滚（有分支/标签兜底）**
    

---

# ✅ Step 0：备份 / 打标（强烈建议）

```bash
git checkout -b rename/welmcli
git tag before-rename-welmcli
```

---

# ✅ Step 1：重命名 GitHub 仓库

在 GitHub：

```
Settings → Repository name → welmcli
```

👉 GitHub 会自动把旧 URL 重定向到新 URL。

---

# ✅ Step 2：更新本地远程地址

```bash
git remote -v           # 先看当前地址
git remote set-url origin <新仓库URL>
git remote -v           # 再确认
```

---

# ✅ Step 3：重命名本地目录

在项目外层执行：

```bash
cd ..
mv clichatgpt welmcli
cd welmcli
```

确认 Git 正常：

```bash
git status
```

---

# ✅ Step 4：全局搜索旧名称（找“写死”的地方）

```bash
grep -R "clichatgpt" .
```

重点检查：

- 硬编码路径（如 `/Users/.../clichatgpt/...`）
    
- bin 脚本名
    
- install.sh / dist
    
- README 示例
    
- 日志前缀（可选是否保留）
    

---

# ✅ Step 5：重命名 CLI 命令为 `welm`

## 5.1 重命名入口脚本

例如：

```bash
mv bin/clichatgpt bin/welm
chmod +x bin/welm
```

## 5.2 更新脚本内部名称（如 usage/日志/帮助）

把：

```bash
clichatgpt ...
```

改为：

```bash
welm ...
```

---

# ✅ Step 6：安装/发布路径调整（如果有）

例如 install.sh 里：

```bash
# 旧
cp dist/clichatgpt /usr/local/bin

# 新
cp dist/welm /usr/local/bin
```

---

# ✅ Step 7（可选但推荐）：保留兼容命令

短期内避免“断崖式切换”：

```bash
# bin/clichatgpt（兼容壳）
#!/usr/bin/env bash
exec welm "$@"
```

👉 这样旧命令还能用，但实际走新实现。

---

# ✅ Step 8：更新 README / 文档

- 项目名：**welmcli**
    
- 命令示例：全部改为 **welm**
    
- 简介一句话（建议）：
    

```text
welmcli — a CLI for steering web interaction flows via CDP
```

---

# ✅ Step 9：自测（必须做）

```bash
# 本地直接跑
./bin/welm --help

# 关键流程
./bin/welm tab find --url chatgpt.com
./bin/welm input '[contenteditable]' "hello"
./bin/welm click '[data-testid=send-button]'
```

---

# ✅ Step 10：提交并推送

```bash
git add -A
git commit -m "rename project to welmcli, cli to welm"
git push origin rename/welmcli
```

如果这是主线：

```bash
git checkout main
git merge rename/welmcli
git push
```

---

# 🧪 快速自检清单（照这个过一遍）

```bash
# 名称残留检查
grep -R "clichatgpt" .

# 命令是否可执行
which welm
welm --help

# 远程是否正确
git remote -v
```

---

# 🧯 回滚方案（万一不满意）

```bash
git checkout before-rename-welmcli
# 或
git reset --hard before-rename-welmcli
```

（GitHub 仓库名也可以再改回去）
